# 安全场景

mmLock 的安全目标是缩短用户离开后的暴露窗口。它适合放在终端、办公设备或智能空间里，作为一个感知触发器：当系统判断用户离开，就可以触发锁屏、隐藏敏感界面或进入保护状态。

## 威胁场景

典型场景是用户离开座位，但设备还没锁定。附近的人可能在这段时间看到屏幕、操作界面或访问数据。

mmLock 关注的是“用户离开”这个行为信号。它不负责替代密码、权限系统或身份认证。更合理的定位是：为已有安全机制提供一个更及时的触发条件。

## 摄像头之外的感知路线

摄像头可以观察人体，但也会采集可见光图像。毫米波雷达提供的是反射信号和点云表示，不直接记录脸、屏幕或房间纹理。对一些设备安全场景来说，这种感知方式更容易放进隐私敏感环境。

这不表示雷达数据就可以随意处理。点云和行为轨迹仍然是感知数据，需要有明确的保存、使用和删除策略。

## 误报和漏报

安全系统不能只看平均准确率。

- 误报：用户还在，系统却触发保护，体验会变差。
- 漏报：用户已经离开，系统没有保护，安全风险更高。

所以站点后续写实验时，应把单用户、附近攻击者、离开角度、位置、速度和环境变化分开讨论。
